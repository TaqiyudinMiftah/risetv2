# CAER-S Data And Protocol Audit

Audit date: 2026-07-14. This report distinguishes the historical community-upstream detector protocol from the clean protocol required for new CD-ICA-Net experiments.

## Decision

The historical detector split is retained only to reproduce earlier CAER-Net runs. It is not valid for controlled research because exact image content overlaps train/validation/test. New runs must use `caer_s_content_disjoint_v1` generated by `prepare_content_disjoint_split.py`.

Validation provenance is resolved: the upstream loader maps logical `train` to physical `CAER-S/train/`, and both logical `val` and `test` to physical `CAER-S/test/`. The notebook previously mapped validation to `train/`; this audit corrected the parser and added a blocking provenance check. Generated clean manifests now normalize this path explicitly.

## Leakage And Duplicates

The source protocol contains 34 duplicate samples within train and 3 within test. Across splits it contains 6 train-validation, 13 train-test, and 2 validation-test overlapping SHA-256 content hashes. These are hash counts; one hash can correspond to multiple rows.

The clean protocol retains the first occurrence in `train -> val -> test` order. Independent audit confirms:

- 69,706 rows and 69,706 unique content hashes;
- zero duplicates within every split;
- zero train-validation, train-test, or validation-test hash overlap;
- detector and manifest order match for every split.

This is exact byte-content disjointness, not video-source or perceptual near-duplicate disjointness. Similar adjacent frames from the same TV source may remain; NCAER-S or a video-identity-aware split is therefore an optional natural-shift check, not a replacement for the frozen main protocol.

## Bounding Boxes And Transforms

The detector files are the community reproduction's cached results from dlib's CNN face detector. Every retained sample has one positive-area box. Of 69,706 boxes, 333 extend outside the image; they are intentionally retained because the upstream loader uses `PIL.Image.crop`, which pads out-of-bounds regions.

Both the notebook and community-upstream code use a `96x96` face, context resize to `128x171`, random `112x112` context crop for training, center crop for validation/test, and ImageNet normalization. The ICCV paper additionally mentions flip, contrast, and color augmentation, which is absent from the available community implementation. Consequently, this track is an upstream community reproduction, not verified official author code.

## Required Use

Generate ignored artifacts with `python prepare_content_disjoint_split.py`, then pass `--detector-dir artifacts/protocols/caer_s_content_disjoint_v1` to `run_caer_official.py`. Do not resume a historical checkpoint on this split or use test metrics for tuning. The next compute-bearing task is the clean CAER-Net exploratory run; it requires an explicit compute decision under the research stop conditions.
